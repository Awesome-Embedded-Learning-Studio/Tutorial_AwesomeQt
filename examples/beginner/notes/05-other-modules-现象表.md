# 05-other-modules 代码样例现象表

> 以下为各示例运行时的预期现象描述，供逐个核实时对照。

## 01-qtsql-database-beginner

| 序号 | 演示要点 | 预期现象 |
|------|----------|----------|
| 1 | 数据库连接与建表（demonstrateConnectionAndSchema） | 控制台输出：`=== 1. 数据库连接与建表 ===`，随后打印 `数据库连接成功`、`驱动名称: QSQLITE`、`支持事务: true`、`employees 表就绪`；在当前目录生成 `company.db` 文件 |
| 2 | 基础 SQL 操作（demonstrateBasicQueries） | 控制台输出：`=== 2. 基础 SQL 操作 ===`，打印 `插入 3 条记录`；然后按薪资降序列出 3 名员工：Carol(92000)、Alice(85000)、Bob(65000)；接着打印更新 Alice 薪资受影响行数 1、删除 Bob 受影响行数 1 |
| 3 | 预编译与参数绑定（demonstratePreparedStatements） | 控制台输出：`=== 3. 预编译与参数绑定 ===`；打印 `命名占位符插入成功（含特殊字符 O'Brien）`；打印 `位置占位符插入成功`；查询 Engineering 部门薪资>75000 的员工并打印结果（Carol、Alice、O'Brien） |
| 4 | 错误处理（demonstrateErrorHandling） | 控制台输出：`=== 4. 错误处理 ===`；打印查询不存在表的错误类型和描述（如 `no such table: nonexistent_table`）；打印违反 NOT NULL 约束的错误信息 |
| 5 | 事务管理（demonstrateTransaction） | 控制台输出：`=== 5. 事务管理（批量插入） ===`；打印 `事务提交成功，批量插入 50 条记录` |
| 6 | 综合查询（demonstrateSummary） | 控制台输出：`=== 6. 综合查询 ===`；按部门分组统计，打印各部门人数和平均薪资；打印总记录数（应为 54 条） |
| 7 | 要点回顾 | 控制台打印 5 条要点回顾，最后输出 `数据库连接已关闭`。程序为纯控制台应用（QCoreApplication），运行后自动退出 |
